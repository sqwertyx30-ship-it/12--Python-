12-недельная программа обучения Python (для начинающего)

Формат: 5 дней изучения + 1 день проект + 1 день повтор/алгоритмы. Рекомендация: 45–90 минут в день.

Неделя 1 — Введение и синтаксис
- Дни 1–3: установка Python, запуск скрипта, `print`, `input`, переменные, типы (`int`, `float`, `str`, `bool`), приведение типов.
- День 4: арифметика, операторы сравнения, `if` / `elif` / `else`.
- День 5: строки, f-строки, базовые методы строк.
- Суббота: мини‑проект — консольный калькулятор.
- Воскресенье: повтор + 15 задач на Codewars/LeetCode (уровень easy).

Неделя 2 — Коллекции
- Дни 1–3: списки, индексация, срезы, методы (`append`, `pop`, `sort`).
- День 4: кортежи, множества.
- День 5: словари, перебор `for k, v in d.items()`.
- Суббота: проект — адресная книга (CRUD в памяти).
- Воскресенье: повтор + упражнения.

Неделя 3 — Циклы и comprehensions
- Дни 1–3: `for`, `while`, `break`, `continue`.
- День 4: list/dict/set comprehensions.
- День 5: генераторы и `yield` (введение).
- Суббота: проект — парсер простого CSV.
- Воскресенье: повтор + задачи на логику.

Неделя 4 — Функции и область видимости
- Дни 1–3: `def`, аргументы, `return`, `None`.
- День 4: `*args`, `**kwargs`, параметры по умолчанию.
- День 5: лямбда, docstrings, аннотации типов (базово).
- Суббота: библиотека вспомогательных функций.
- Воскресенье: повтор + рефакторинг.

Неделя 5 — Работа с файлами и исключения
- Дни 1–3: `open`, режимы `r/w/a`, `with`.
- День 4: CSV и JSON (`csv`, `json`).
- День 5: `try/except/finally`, логирование (`logging`).
- Суббота: проект — логгер/запись событий.
- Воскресенье: повтор + документация проекта.

Неделя 6 — Модули, окружения, Git
- Дни 1–2: модули и пакеты, `__name__ == "__main__"`.
- День 3: `venv`, `pip`, `requirements.txt`.
- День 4: структура проекта Python.
- День 5: основы Git (init, add, commit, push).
- Суббота: упаковать мини‑проект и загрузить на GitHub.
- Воскресенье: повтор + README.

Неделя 7 — ООП
- Дни 1–3: классы, `__init__`, методы, атрибуты.
- День 4: наследование, полиморфизм.
- День 5: магические методы (`__str__`, `__repr__`, `__eq__`).
- Суббота: проект — текстовая игра или модель объектов.
- Воскресенье: повтор + тесты.

Неделя 8 — Тестирование и инструменты качества
- Дни 1–2: `unittest`/`pytest` — писать тесты.
- День 3: отладка (`pdb`), логирование.
- День 4: `black`, `flake8` — форматирование и линтинг.
- День 5: непрерывная интеграция (шаблон GitHub Actions).
- Суббота: добавить тесты и CI к проекту.
- Воскресенье: повтор + исправления.

Неделя 9 — Работа с сетью и API
- Дни 1–2: `requests`, GET/POST, обработка JSON.
- День 3: парсинг HTML (`BeautifulSoup`).
- День 4: основы REST API, авторизация.
- День 5: практическая задача с публичным API.
- Суббота: проект — клиент погоды или другой API.
- Воскресенье: повтор + документация API.

Неделя 10 — Базы данных
- Дни 1–2: SQLite (`sqlite3`), CRUD.
- День 3: SQL (SELECT, INSERT, UPDATE, DELETE).
- День 4: ORM (введение в SQLAlchemy или Django ORM).
- День 5: интеграция БД в проект.
- Суббота: проект — TODO с хранением в SQLite.
- Воскресенье: повтор + экспорт/бэкап.

Неделя 11 — Выбор направления
- Вариант A: Веб — `Flask`/`Django`, шаблоны, формы, деплой.
- Вариант B: Data — `pandas`, `numpy`, визуализация.
- Выбери одно направление, работай над проектом недели.

Неделя 12 — Финальный проект и портфолио
- Дни 1–3: реализация проекта (функционал).
- День 4: тесты, документация, README.
- День 5: деплой/публикация на GitHub.
- Суббота: подготовка резюме и краткого описания проектов.
- Воскресенье: обратная связь, доработка по отзывам.

Советы
- Коммить часто, маленькими шагами.
- Пиши README и инструкции запуска для каждого проекта.
- Решай короткие задачи регулярно (20–30 минут).
- Проси ревью кода — это ускоряет рост.
